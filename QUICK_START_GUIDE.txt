## 🚀 快速启动步骤

### 步骤1: 启动Docker服务
```bash
cd E:\大模型开发\代码\网站\travel_proj
docker-compose -f docker-compose-milvus.yml up -d
```

**启动的服务：**
- etcd (Milvus依赖)
- minio (对象存储) - 端口9000, 9001
- milvus (向量数据库) - 端口19530, 9091

**验证：**
```bash
docker ps | grep -E "minio|milvus|etcd"
```

### 步骤2: 启动后端
```bash
cd backend
python main.py
```

**观察健康检查日志：**
```
[健康检查] 检查服务连接...
[1/4] ✓ Milvus连接成功 (当前向量数: 0)
[2/4] ✓ Neo4j连接成功 (当前节点数: 139)
[3/4] ✓ Redis连接成功
[4/4] ✓ 存储可用
[健康检查] 所有服务连接正常 ✓
```

### 步骤3: 启动前端
```bash
cd frontend
npm run dev
```

**访问：** `http://localhost:5173`

## 🔧 常见问题

### 问题1: Milvus连接失败
**错误：** `Failed to connect to Milvus`

**解决：**
```bash
# 检查Milvus是否运行
docker ps | grep milvus

# 如果没有，启动它
docker start milvus-standalone

# 等待30秒
```

### 问题2: Neo4j连接失败
**错误：** `Neo4j connection failed`

**解决：**
```bash
# 检查Neo4j容器
docker ps | grep neo4j

# 或检查本地Neo4j服务
# 访问 http://localhost:7474
```

### 问题3: Redis连接失败
**错误：** `Redis connection failed`

**解决：**
```bash
# 检查Redis
docker ps | grep redis

# 或启动Redis
docker start travel_redis
```

### 问题4: MinIO连接失败
**错误：** `MinIO connection failed`

**解决：**
```bash
# 检查MinIO
docker ps | grep minio

# 启动MinIO
docker start milvus-minio

# 等待10秒
```

## 🎯 配置文件

### 后端配置
**文件：** `backend/.env`

**关键配置：**
```env
# API
API_HOST=0.0.0.0
API_PORT=8000

# Milvus
MILVUS_HOST=localhost
MILVUS_PORT=19530

# Neo4j
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=..123456

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# MinIO
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=travel-documents
MINIO_SECURE=false
```

## 🔄 重启所有服务

```bash
# 停止所有服务
docker-compose -f docker-compose-milvus.yml down

# 启动所有服务
docker-compose -f docker-compose-milvus.yml up -d

# 等待30秒
sleep 30

# 重启后端
cd backend
python main.py

# 刷新前端
Ctrl + Shift + R
```
