## 🔍 问题原因

**错误信息：**
```
expect 13 list, got 12
```

**分析：**
- 新代码的Schema定义了12个字段
- insert方法传入了12个字段
- 但Milvus中已存在的collection有13个字段（旧版本）

**结论：** 需要删除旧collection并重建

---

## ✅ 解决方案

### 方案1: Python脚本删除（推荐）

**在backend目录执行：**
```bash
cd backend

python -c "
import sys
sys.path.insert(0, '.')

from pymilvus import connections, utility

# 连接Milvus
connections.connect(
    alias='default',
    host='localhost',
    port=19530
)

# 删除旧collection
collection_name = 'travel_destinations'

if utility.has_collection(collection_name):
    utility.drop_collection(collection_name)
    print(f'✓ 已删除collection: {collection_name}')
else:
    print(f'× Collection不存在: {collection_name}')

print('✓ 完成，现在可以重启后端')
"
```

---

### 方案2: 修改collection名称

**修改 `.env` 文件：**
```env
# 原来
MILVUS_COLLECTION_NAME=travel_destinations

# 改为
MILVUS_COLLECTION_NAME=travel_destinations_v2
```

这样会创建新的collection，避免冲突。

---

### 方案3: 重启Milvus服务

如果使用Docker：
```bash
docker restart milvus
```

如果使用standalone：
```bash
docker-compose down
docker-compose up -d
```

## ⚠️ 注意事项

**删除collection会：**
- ✓ 清空所有已存储的向量数据
- ✓ 重新开始，Schema正确
- ✗ 丢失之前上传的数据

如果需要保留数据，使用方案2（改名）。
