## 问题1: MinerU解析文件没有上传到MinIO

### 📋 问题描述
MinIO中只有`阿里攻略（第六版）_origin.pdf`，没有MinerU解析后的Markdown文件。

### 🔍 原因分析
查看node_minio.py代码：
```python
# 4. 检查是否有MinerU解析结果
mineru_result = state.get('mineru_result')
if mineru_result:
    # 保存Markdown
```

**问题：** state中没有`mineru_result`字段！

实际上PDF解析节点保存的是：
- `state['parsed_text']` - Markdown文本
- `state['markdown_text']` - Markdown文本
- 但没有`state['mineru_result']`

### ✅ 解决方案

**修改文件：** `backend/app/workflows/document_processing/nodes/node_minio.py`

```python
# 修改前
mineru_result = state.get('mineru_result')
if mineru_result:
    markdown_text = state.get('parsed_text')

# 修改后
markdown_text = state.get('parsed_text') or state.get('markdown_text')
if markdown_text:
    # 保存Markdown到临时文件
    import tempfile
    with tempfile.NamedTemporaryFile(mode='w', encoding='utf-8', suffix='.md', delete=False) as f:
        f.write(markdown_text)
        temp_md_path = f.name

    # 上传Markdown文件
    md_object_name = f"{name_without_ext}/{name_without_ext}.md"
    logger.info(f"[MinIO] Uploading parsed Markdown: {md_object_name}")
    minio.save_file(temp_md_path, md_object_name)
    logger.success(f"[MinIO] Markdown uploaded: {md_object_name}")

    # 清理临时文件
    os.remove(temp_md_path)
```

**关键修改：**
1. 移除`mineru_result`检查
2. 直接检查`parsed_text`或`markdown_text`
3. 如果有内容，就保存为.md文件并上传

## 问题3: JSON解析错误

### 📋 问题描述
```
[Extract] JSON parse failed for chunk 9: Extra data: line 134 column 3 (char 3142)
```

### 🔍 原因分析
Qwen API返回的JSON中包含多余的数据：
- 可能返回了多个JSON对象
- 或者JSON后面有额外的文本

### ✅ 解决方案

**修改文件：** `backend/app/workflows/document_processing/nodes/node_entity_extraction.py`

**方案1: 使用JSONDecoder解析多个JSON对象**

```python
import json
from json import JSONDecoder

def extract_json_objects(text):
    """从文本中提取所有JSON对象"""
    decoder = JSONDecoder()
    objects = []
    idx = 0

    while idx < len(text):
        text = text[idx:].lstrip()
        if not text:
            break
        try:
            obj, end_idx = decoder.raw_decode(text)
            objects.append(obj)
            idx += end_idx
        except json.JSONDecodeError:
            break

    return objects

# 在解析时使用
try:
    # 尝试标准解析
    result = json.loads(response_text)
except json.JSONDecodeError as e:
    # 如果失败，尝试提取多个JSON对象
    logger.warning(f"[Extract] Standard JSON parse failed, trying multi-object parse")
    objects = extract_json_objects(response_text)
    if objects:
        # 合并所有对象的destinations
        result = {"destinations": []}
        for obj in objects:
            if isinstance(obj, dict) and "destinations" in obj:
                result["destinations"].extend(obj["destinations"])
    else:
        raise e
```

**方案2: 清理响应文本**

```python
# 在解析前清理文本
def clean_json_response(text):
    """清理JSON响应"""
    # 移除markdown代码块标记
    text = text.strip()
    if text.startswith("```json"):
        text = text[7:]
    if text.startswith("```"):
        text = text[3:]
    if text.endswith("```"):
        text = text[:-3]

    # 尝试找到第一个{和最后一个}
    start = text.find('{')
    end = text.rfind('}')
    if start != -1 and end != -1:
        text = text[start:end+1]

    return text.strip()

# 使用
response_text = clean_json_response(response_text)
result = json.loads(response_text)
```

**方案3: 增加重试机制**

```python
# 如果解析失败，记录但继续处理
try:
    result = json.loads(response_text)
except json.JSONDecodeError as e:
    logger.warning(f"[Extract] JSON parse failed for chunk {i}: {e}")
    logger.debug(f"[Extract] Raw response: {response_text[:500]}")
    # 跳过这个chunk，继续处理下一个
    continue
```
