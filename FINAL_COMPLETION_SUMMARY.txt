## ⚠️ pages_count为0的原因

### 可能原因1: PyPDF2读取失败
PDF解析节点中使用PyPDF2获取页数，可能失败并捕获了异常。

### 可能原因2: State没有传递
PDF解析节点设置了`state['pages_count']`，但可能没有正确传递到finalize。

---

## 🔧 完整修复方案

### 修复1: 增强PDF解析节点的页数获取
**文件：** `backend/app/workflows/document_processing/nodes/node_pdf_parse.py`

```python
# Get PDF page count
pages_count = 0
try:
    import PyPDF2
    with open(file_path, 'rb') as pdf_file:
        pdf_reader = PyPDF2.PdfReader(pdf_file)
        pages_count = len(pdf_reader.pages)
        logger.info(f"[PDF] PDF页数: {pages_count}")

        # Store page count in state
        state['pages_count'] = pages_count
except Exception as e:
    logger.error(f"[PDF] 无法获取页数: {e}")
    state['pages_count'] = 0
```

### 修复2: 确保finalize获取pages_count
**文件：** `backend/app/workflows/document_processing/nodes/node_finalize.py`

```python
# Get page count from state
pages_count = state.get('pages_count', 0)

# Update Redis
task_info['pages_count'] = pages_count

logger.success(f"[Finalize] Updated: {destinations_count} destinations, {pages_count} pages")
```

### 修复3: 添加upload_time字段
**文件：** `backend/app/services/document_task.py`

```python
task_info = {
    ...
    "upload_time": datetime.now().isoformat()
}
```
