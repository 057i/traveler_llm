## 🐛 问题诊断

### 发现的问题

1. **后端缺少document_id字段**
   - 前端期望: `response.document_id` 和 `response.task_id`
   - 后端返回: 只有 `task_id`
   - 导致: 前端无法关联文档和任务

2. **前端显示"无处理进度信息"**
   - 原因: `row.taskId` 为空
   - 条件: `v-if="row.taskId && (row.status === 'processing' || row.status === 'pending')"`
   - 结果: WorkflowProgress组件未渲染

## ✅ 修复内容

### 后端修复 (documents.py)

**修改前:**
```python
return {
```

### 前端逻辑 (DocumentManager.vue)

**上传成功处理:**
```javascript
const handleUploadSuccess = (response, file) => {
  // 1. 显示成功消息
  ElMessage.success(`${file.name} 上传成功！`)

  // 2. 刷新文档列表
  setTimeout(async () => {
    await fetchDocuments()

    // 3. 关联taskId到文档
    if (response.document_id && response.task_id) {
      const doc = documents.value.find(d => d.id === response.document_id)
      if (doc) {
        doc.taskId = response.task_id  // 关键：设置taskId
        // 4. 自动展开显示进度
        if (!expandedRows.value.includes(doc.id)) {
          expandedRows.value.push(doc.id)
        }
      }
    }
  }, 100)
}
```

**进度显示条件:**
```vue
<template>
  <div v-if="row.taskId && (row.status === 'processing' || row.status === 'pending')"
       class="progress-detail">
    <!-- 显示WorkflowProgress -->
    <WorkflowProgress
      :task-id="row.taskId"
      :filename="row.filename"
      @complete="handleProcessComplete"
    />
  </div>
  <div v-else class="no-progress">
    <!-- 显示"无处理进度信息" -->
    <el-empty description="无处理进度信息" :image-size="60" />
  </div>
</template>
```

## 🔄 完整工作流程

```
1. 用户点击上传PDF
   ↓
2. 前端发送POST /api/documents/upload
   ↓
3. 后端处理
   • 生成task_id (UUID)
   • 保存文件
   • 创建任务
   • 启动后台处理
   • 返回: {task_id, document_id, filename, status}
   ↓
4. 前端收到响应
   • handleUploadSuccess(response, file)
   • 100ms后刷新文档列表
   ↓
5. 关联taskId
   • 查找doc: documents.find(d => d.id === response.document_id)
   • 设置: doc.taskId = response.task_id
   • 展开: expandedRows.push(doc.id)
   ↓
6. 渲染WorkflowProgress组件
   • 接收props: task-id, filename
   • 连接SSE: /api/documents/progress/{taskId}
   ↓
7. SSE实时推送进度
   • event: start
   • event: progress (6次，每个节点)
   • event: step_complete (6次)
   • event: complete
   ↓
8. 前端显示进度
   • 进度条: 0% → 15% → 35% → 50% → 65% → 80% → 95% → 100%
   • 节点状态: pending → processing → completed
   • 节点名称: MinIO存储, PDF解析, 文本分块, 实体提取, RAG向量化, 知识图谱构建
```

## 📊 SSE事件格式

### 后端发送

```javascript
// 开始事件
event: start
data: {"message": "开始处理文档: xxx.pdf", "filename": "xxx.pdf"}

// 进度事件
event: progress
data: {
  "step": "minio",
  "step_name": "MinIO存储",
  "status": "processing",
  "message": "正在保存文件到存储...",
  "progress": 15
}

// 步骤完成
event: step_complete
data: {
  "step": "minio",
  "step_name": "MinIO存储",
  "status": "completed",
  "message": "文件保存成功",
  "progress": 15
}

// 全部完成
event: complete
data: {
  "message": "处理完成: 提取了 8 个景点",
  "destinations_count": 8,
  "progress": 100
}
```

### 前端监听

```javascript
eventSource.addEventListener('progress', (event) => {
  const data = JSON.parse(event.data)

  // 更新进度条
  if (data.progress !== undefined) {
    progress.value = data.progress
  }

  // 更新节点状态
  if (data.step) {
    updateStepStatus(data.step, 'processing')
    statusText.value = data.message
  }
})

eventSource.addEventListener('step_complete', (event) => {
  const data = JSON.parse(event.data)

  // 标记节点完成
  if (data.step) {
    updateStepStatus(data.step, 'completed')
  }
})
```
