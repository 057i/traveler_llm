## 🔍 问题现象

上传PDF后，文档显示"完成"但没有调用SSE，没有显示处理进度。

## 📋 可能的原因

### 原因1：时序问题 - 处理太快 ⚡

**问题：**
- 后台任务处理非常快（几秒内完成）
- 前端 `setTimeout(100ms)` 刷新列表时，文档已经处理完成
- 文档状态已经是 `"completed"` 而不是 `"processing"`
- 前端条件 `v-if="row.status === 'processing'"` 不满足

**验证方法：**
查看后端日志，从 `[Background] Starting` 到 `[Workflow] Completed` 的时间

---

### 原因2：taskId关联失败 ❌

**问题：**
- `response.document_id` 和列表中的 `d.id` 不匹配
- `find(d => d.id === response.document_id)` 找不到文档
- `doc.taskId` 没有设置成功

**验证方法：**
浏览器控制台查看 `response` 和 `documents.value`

---

### 原因3：文档API返回的id格式问题 🔢

**问题：**
- 后端使用 `task_id` 作为 `document_id`
- 但数据库中的文档可能有不同的 `id`
- 导致 `find()` 找不到匹配的文档

---

## ✅ 解决方案

### 方案1：修改前端显示条件（推荐）

**问题：** 条件太严格，只在 `processing` 状态显示进度

**解决：** 只要有 `taskId` 就显示进度组件

```vue
<!-- 修改前 -->
<div v-if="row.taskId && (row.status === 'processing' || row.status === 'pending')"
     class="progress-detail">
  <WorkflowProgress ... />
</div>

<!-- 修改后 -->
<div v-if="row.taskId" class="progress-detail">
  <WorkflowProgress ... />
</div>
```

**优点：**
- 即使处理快，也能看到完成的进度
- WorkflowProgress组件会处理已完成的状态

---

### 方案2：强制设置状态为processing

**在 handleUploadSuccess 中：**

```javascript
if (response.document_id && response.task_id) {
  const doc = documents.value.find(d => d.id === response.document_id)
  if (doc) {
    doc.taskId = response.task_id
    doc.status = 'processing'  // ← 强制设置为processing

    if (!expandedRows.value.includes(doc.id)) {
      expandedRows.value.push(doc.id)
    }
  }
}
```

---

### 方案3：添加调试日志

**在 handleUploadSuccess 中添加：**

```javascript
const handleUploadSuccess = (response, file) => {
  console.log('[Upload] Response:', response)

  ElMessage.success(`${file.name} 上传成功！`)

  uploading.value = false
  uploadDialogVisible.value = false
  clearFiles()

  setTimeout(async () => {
    await fetchDocuments()

    console.log('[Upload] Documents after fetch:', documents.value)
    console.log('[Upload] Looking for document_id:', response.document_id)

    if (response.document_id && response.task_id) {
      const doc = documents.value.find(d => d.id === response.document_id)

      console.log('[Upload] Found doc:', doc)

      if (doc) {
        doc.taskId = response.task_id
        doc.status = 'processing'

        console.log('[Upload] Set taskId:', doc.taskId)
        console.log('[Upload] Set status:', doc.status)

        if (!expandedRows.value.includes(doc.id)) {
          expandedRows.value.push(doc.id)
        }
      } else {
        console.error('[Upload] Document not found! document_id:', response.document_id)
      }
    }

    fetchStatistics()
  }, 100)
}
```

### 方案5：修改后端，确保返回数据库ID

**如果问题是ID不匹配，修改后端：**

```python
# documents.py
@router.post("/upload")
async def upload_document(file: UploadFile):
    task_id = str(uuid.uuid4())

    # 保存文件...

    # 创建数据库记录（如果有）
    # db_doc = await db.create_document(...)
    # document_id = db_doc.id

    return {
        "task_id": task_id,
        "document_id": task_id,  # 确保这个ID能在列表中找到
        "filename": file.filename,
        "status": "processing"
    }
```

### 步骤3：检查网络请求

**开发者工具 → 网络标签**

1. 找到 `/api/documents/upload` 请求
   - 查看响应是否包含 `document_id` 和 `task_id`

   - 如果有但404 = taskId有问题
   - 如果有但200 = SSE正常，可能是前端显示问题

---

### 步骤4：检查后端日志

查找以下日志：

```
[INFO] 文件上传完成: xxx.pdf, 任务ID: abc-123
[INFO] [Background] Starting document processing for task: abc-123  ← 是否有这行
[INFO] [Workflow] Starting: xxx.pdf  ← 是否有这行
```

**如果没有 [Background] 日志：**
- 后台任务未启动

**如果有完整流程但很快完成：**
- 时序问题，用方案1或方案2
