## 🐛 根本原因

### 代码分析 (workflow_service.py:125)
```python
if step_id and step_id not in completed_steps:
    # 发送SSE事件
    await self._send_step_event(...)
    completed_steps.add(step_id)
```

**问题：**
- 每个step_id只发送一次事件
- 如果fallback节点先执行，主节点会被跳过
- 或者节点执行太快，事件被合并

---

## ✅ 解决方案

### 方案1: 调试日志（立即实施）
添加日志确认SSE事件是否发送：

```python
# workflow_service.py:125
if step_id and step_id not in completed_steps:
    logger.info(f"[SSE] Sending events for step: {step_id}")
    await self._send_step_event(...)
else:
    logger.warning(f"[SSE] Skipping step (already completed): {step_id}")
```

### 方案2: 移除去重逻辑（推荐）
允许相同step_id多次发送事件：

```python
if step_id:
    # 移除 "not in completed_steps" 检查
    step_name = self.STEP_NAMES.get(step_id, step_id)

    # 只有当状态真的变化时才发送
    if step_id not in completed_steps:
        await self._send_step_event(status="processing")
        await asyncio.sleep(0.1)
        await self._send_step_event(status="completed")
        completed_steps.add(step_id)
```
