## 📊 最终Milvus Schema

```
destination_id (主键)
├── dense_vector (768维)      # bge-base-zh-v1.5稠密向量
├── sparse_vector (稀疏)       # TF-IDF稀疏向量
├── name                       # 景点名称
├── province                   # 省份
├── city                       # 城市
├── category                   # 分类
├── description                # 描述
├── filename                   # 来源文件名（如：三清山攻略.pdf）
├── entity_json                # 完整实体JSON字符串
├── full_text                  # 完整格式化文本（用于检索）
└── rating                     # 评分
```

## 🔄 完整数据流程

```
1. 用户上传PDF: 三清山攻略.pdf
   ↓
2. MinerU解析 → 提取文本
   ↓
3. 文本分块
   ↓
4. Qwen提取实体 → 28个景点
   ↓
5. 向量化节点 (vectorize_traditional):

   a) 准备文本:
      - 基本文本: "三清山 世界自然遗产... 江西省 上饶市 自然风光 世界遗产"
      - 完整文本: "三清山 | 类型: 景点 | 分类: 山脉 | 位置: 江西省上饶市 | ..."

   b) 生成稠密向量:
      embedding_service.encode(texts) → 768维向量

   c) 生成稀疏向量:
      embedding_service.encode_sparse(texts) → {idx: value}

   d) 准备数据:
      {
        "destination_id": "三清山_江西省上饶市",
        "dense_vector": [0.1, 0.2, ..., 768维],
        "sparse_vector": {0: 0.5, 10: 0.3, ...},
        "name": "三清山",
        "province": "江西省",
        "city": "上饶市",
        "category": "山脉",
        "description": "世界自然遗产，以奇峰怪石、云海日出著称",
        "filename": "三清山攻略.pdf",
        "entity_json": "{\"name\":\"三清山\",\"city\":\"上饶市\",...}",
        "full_text": "三清山 | 类型: 景点 | ...",
        "rating": 4.0
      }

   e) 插入Milvus:
      milvus_hybrid_client.insert(data)
   ↓
6. GraphRAG (vectorize_graph):
   Neo4j存储知识图谱
   ↓
7. 完成
```

## 🎯 混合检索优势

### 稠密向量（Dense Vector）
- **模型：** bge-base-zh-v1.5（本地模型）
- **维度：** 768
- **优势：** 语义理解、同义词、上下文

### 稀疏向量（Sparse Vector）
- **方法：** TF-IDF + jieba分词
- **格式：** {idx: value}
- **优势：** 精确关键词匹配、词频统计

### RRF融合
- **算法：** Reciprocal Rank Fusion
- **参数：** k=60
- **优势：** 结合双向量优势，更准确的排序
