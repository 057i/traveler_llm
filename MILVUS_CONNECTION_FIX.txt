## 🔍 问题确认

**错误：** `Fail connecting to server on localhost:19530, server unavailable`

**原因：** Milvus服务未启动

---

## ✅ 解决方案

### 方案1: 检查并启动Milvus Docker容器

**步骤1: 查看所有容器**
```bash
docker ps -a | grep milvus
```

**步骤2: 如果看到容器，启动它**
```bash
docker start milvus-standalone
```

**或使用docker-compose：**
```bash
docker-compose up -d
```

---

### 方案2: 如果没有Milvus容器

**快速启动Milvus Standalone：**
```bash
# 下载并运行
docker run -d --name milvus-standalone \
  -p 19530:19530 \
  -p 9091:9091 \
  -v milvus_data:/var/lib/milvus \
  milvusdb/milvus:latest
```

**等待30秒让Milvus启动完成**

---

### 方案3: 检查端口占用

```bash
# Windows
netstat -ano | findstr :19530

# 如果端口被占用，找到进程ID并结束
```

## 📝 Milvus快速启动指南

### 使用Docker Compose（推荐）

**创建 `docker-compose.yml`：**
```yaml
version: '3.5'

services:
  milvus:
    container_name: milvus-standalone
    image: milvusdb/milvus:latest
    command: milvus run standalone
    environment:
      ETCD_USE_EMBED: true
      COMMON_STORAGETYPE: local
    volumes:
      - ${DOCKER_VOLUME_DIRECTORY:-.}/volumes/milvus:/var/lib/milvus
    ports:
      - "19530:19530"
      - "9091:9091"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9091/healthz"]
      interval: 30s
      start_period: 90s
      timeout: 20s
      retries: 3
```

**启动：**
```bash
docker-compose up -d
```

## ⚠️ 常见问题

### 问题1: Docker未安装
**症状：** `docker: command not found`
**解决：** 安装Docker Desktop for Windows

### 问题2: 端口被占用
**症状：** `port is already allocated`
**解决：**
```bash
# 查找占用进程
netstat -ano | findstr :19530

# 结束进程或更改端口
```

### 问题3: Milvus启动慢
**症状：** 等待很久仍无法连接
**解决：**
```bash
# 查看日志
docker logs -f milvus-standalone

# 等待看到 "Milvus Proxy successfully started"
```
