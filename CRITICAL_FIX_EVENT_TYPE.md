# 🎯 关键修复：event_type字段名

## 问题根源

**Result事件使用了错误的字段名！**

```python
# 错误
result_event = {
    "type": "result",  # ❌ 错误
    ...
}

# 正确
result_event = {
    "event_type": "result",  # ✅ 正确
    ...
}
```

---

## 为什么？

API从Redis获取事件后，使用以下代码获取事件类型：

```python
event_type = data.get('event_type') or data.get('type')
```

虽然有fallback到'type'，但是：
1. 其他所有事件（node_start, node_end, complete）都使用'event_type'
2. 可能某些检查逻辑只看'event_type'
