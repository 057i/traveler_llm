# 🎯 置信度计算方案对比

## 📊 方案对比

### **方案A: 简单规则打分（旧方案）**

**逻辑**：
```python
confidence = 0
if 精确匹配在Top3: confidence += 0.5
if 结果≥10条: confidence += 0.3
if 分数分布好: confidence += 0.2
```

**优点**：
- ✅ 简单直观
- ✅ 易于调试

**缺点**：
- ❌ 主观性强
- ❌ 阈值难调
- ❌ 无法适应不同查询类型

---

### **方案B: 多维度综合计算（新方案）⭐推荐**

**维度1: 检索质量（40%权重）**
- 精确匹配位置（Top1=0.5, Top3=0.4, Top5=0.3）
- 向量相似度（原始score的归一化）
- Top1来源类型（exact>dense>sparse）

**维度2: 结果覆盖（30%权重）**
- 结果数量（≥10=0.5, ≥5=0.3, ≥3=0.2）
- 来源多样性（3种=0.5, 2种=0.3, 1种=0.1）

**维度3: 排序稳定性（20%权重）**
- Top1突出度（Top1与中位数的相对差距）
- 分数分布方差（适度方差为佳）

**维度4: 查询复杂度（10%权重）**
- 查询长度（≥10字=0.4, ≥5字=0.3）
- 实体数量（≥2=0.4, 1=0.3, 0=0.1）
- 查询类型（精确/模糊）

**优点**：
- ✅ 科学性强，多维度评估
- ✅ 自适应不同查询类型
- ✅ 可解释性好（各维度分数可观察）
- ✅ 可调整权重适配业务

**缺点**：
- ⚠️ 计算稍复杂
- ⚠️ 需要观察日志调优

---

## 🧪 实际效果对比

### **场景1: "三清山"（精确查询）**

**方案A（旧）**：
```
精确匹配Top1: +0.5
结果10条: +0.3
分数分布: +0.15
-------------------
总计: 0.95 ✅
```

**方案B（新）**：
```
检索质量: 0.9  (精确匹配Top1=0.5 + 相似度=0.2 + 来源=0.2)
结果覆盖: 0.8  (数量=0.5 + 多样性=0.3)
排序稳定: 0.7  (Top1突出=0.4 + 方差=0.3)
查询复杂: 0.6  (长度=0.2 + 实体=0.3 + 类型=0.1)
-------------------
总计: 0.4×0.9 + 0.3×0.8 + 0.2×0.7 + 0.1×0.6 = 0.86 ✅
```

**结论**：两种方案都能正确识别高置信度

## 📈 优势分析

### **方案B的核心优势**

#### **1. 自适应查询类型**
```
精确查询（"三清山"）:
  检索质量高(0.9) + 查询简单(0.6) = 高置信度 ✅

模糊查询（"推荐景点"）:
  检索质量中(0.4) + 查询复杂(0.7) = 中等置信度 ⚠️

短查询（"美食"）:
  检索质量低(0.3) + 查询模糊(0.3) = 低置信度 ❌
```

#### **2. 可观测性强**
```
日志输出：
[Confidence] 检索质量: 0.900
[Confidence] 结果覆盖: 0.800
[Confidence] 排序稳定性: 0.700
[Confidence] 查询复杂度: 0.600
[Confidence] 综合置信度: 0.860

👉 可以直接看到哪个维度拖后腿
```

#### **3. 易于调优**
```python
# 根据业务调整权重
weights = {
    "retrieval_quality": 0.4,    # 注重检索质量
    "result_coverage": 0.3,      # 结果充足性次之
    "ranking_stability": 0.2,    # 排序稳定性
    "query_complexity": 0.1      # 查询复杂度影响小
}

# 如果发现Tavily调用过少，可以调整：
weights = {
    "retrieval_quality": 0.5,    # 提高检索质量权重
    "result_coverage": 0.25,     # 降低覆盖权重
    "ranking_stability": 0.15,
    "query_complexity": 0.1
}
```

## 🎯 推荐配置

### **标准配置（平衡型）**
```python
# settings.py
CONFIDENCE_THRESHOLD = 0.6

# confidence_calculator.py
weights = {
    "retrieval_quality": 0.4,
    "result_coverage": 0.3,
    "ranking_stability": 0.2,
    "query_complexity": 0.1
}
```

**预期效果**：
- 精确查询：0.8-0.9 → 不触发Tavily
- 模糊查询：0.5-0.7 → 部分触发Tavily
- 短查询：0.3-0.5 → 触发Tavily

```python
CONFIDENCE_THRESHOLD = 0.7  # 提高阈值

weights = {
    "retrieval_quality": 0.5,   # 更注重检索质量
    "result_coverage": 0.2,
    "ranking_stability": 0.2,
    "query_complexity": 0.1
}
```

### **激进配置（节省Tavily）**
```python
CONFIDENCE_THRESHOLD = 0.5  # 降低阈值

weights = {
    "retrieval_quality": 0.3,
    "result_coverage": 0.4,     # 更注重结果数量
    "ranking_stability": 0.2,
    "query_complexity": 0.1
}
```
