## 🚀 使用步骤

### 步骤1: 安装minio包
```bash
cd backend
pip install minio
```

### 步骤2: 启动MinIO服务
```bash
cd E:\大模型开发\代码\网站\travel_proj
docker-compose -f docker-compose-milvus.yml up -d
```

**包含的服务：**
- etcd (Milvus依赖)
- minio (对象存储)
- milvus (向量数据库)

### 步骤3: 验证MinIO启动
```bash
docker ps | grep minio
```

**应该看到：**
```

```

打开浏览器访问：`http://localhost:9001`

**登录信息：**
- 用户名: `minioadmin`
- 密码: `minioadmin`

```bash
cd backend
python main.py
```

**观察启动日志：**

## ⚙️ 配置说明

### MINIO_ENDPOINT
MinIO服务器地址，格式：`host:port`
- 本地开发：`localhost:9000`
- 生产环境：`minio.yourdomain.com:9000`

### MINIO_ACCESS_KEY / MINIO_SECRET_KEY
MinIO访问凭证
- 默认：`minioadmin` / `minioadmin`
- 生产环境：使用强密码

### MINIO_BUCKET
存储桶名称
- 必须符合S3命名规范（小写、连字符）
- 会自动创建

### MINIO_SECURE
是否使用HTTPS
- `false`: http://
- `true`: https://

## 🐛 常见问题

### 问题1: 连接失败
**错误：** `[MinIO] Connection failed`

**解决：**
1. 检查MinIO是否运行：`docker ps | grep minio`
2. 检查端口是否开放：`netstat -ano | findstr 9000`
3. 检查配置是否正确

### 问题2: Bucket不存在
**错误：** `Bucket does not exist`

**解决：**
客户端会自动创建bucket，如果失败：
1. 检查权限
2. 手动在控制台创建

### 问题3: 上传失败
**错误：** `Upload failed`

**解决：**
1. 检查磁盘空间
2. 检查文件权限
3. 查看MinIO日志：`docker logs milvus-minio`

## 📝 迁移旧文件（可选）

如果需要迁移本地存储的旧文件到MinIO：

```python
# 创建迁移脚本
import os
from app.core.minio_client import get_minio_client

minio = get_minio_client()
local_dir = "./data/uploads"

for filename in os.listdir(local_dir):
    file_path = os.path.join(local_dir, filename)
    minio.save_file(file_path, filename)
    print(f"Migrated: {filename}")
```
