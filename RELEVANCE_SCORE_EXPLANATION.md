# 📊 参考来源相关度分值计算说明

## 计算流程

### **1. 混合检索阶段**
```
查询: "推荐三清山"

↓ Dense检索 (语义相似度)
  - 使用 bge-base-zh-v1.5 模型
  - 计算查询和文档的向量相似度
  - 返回 Top 9 结果

↓ Sparse检索 (关键词匹配)
  - 使用 BM25 算法
  - 基于词频和逆文档频率
  - 返回 Top 10 结果

↓ RRF融合 (Reciprocal Rank Fusion)
  - 融合Dense和Sparse的结果
  - 计算综合排名分数
  - 返回 Top 15 结果
```

### **2. Rerank重排序阶段**
```python
# 使用 bge-reranker-v2-m3 模型
# 对每个候选文档进行深度语义相关度评分

model.predict([
    ["推荐三清山", "玉京峰景区 三清山最高峰..."],
    ["推荐三清山", "女神峰 端庄秀丽..."],
    ["推荐三清山", "三清山 道教名山..."]
])

# 返回分数（CrossEncoder输出）:
# [0.5012, 1.1234, 0.7456]
```

### **3. 前端显示**
```javascript

```

## 分数含义

### **Rerank分数范围**
- **> 1.0** (>100%)：高度相关，语义完全匹配
- **0.8 - 1.0** (80-100%)：强相关
- **0.5 - 0.8** (50-80%)：相关
- **0.3 - 0.5** (30-50%)：弱相关
- **< 0.3** (<30%)：不太相关

### **为什么会超过100%？**
Rerank模型（CrossEncoder）输出的**不是概率**，而是**相似度得分**：
- 理论上没有上限
- 分数越高表示语义相关度越强
- 通常范围在 -1 到 +2 之间
- 正常情况下，最相关的结果可以达到 1.5-2.0

---

## 技术细节

### **BGE-Reranker-v2-m3 模型**
- **类型**: CrossEncoder (交叉编码器)
- **输入**: [查询, 文档] 成对输入
- **输出**: 相似度分数 (实数)
- **优势**: 比双塔模型（Dense检索）更准确

### **计算方式**
```python
# CrossEncoder内部计算
score = model([query, document])

# 实际是：
# 1. 将query和document拼接
# 2. 通过BERT模型编码
# 3. 通过分类层输出相似度分数
# 4. 分数无上下限（但通常在-1到2之间）
```

## 示例

### **查询**: "推荐三清山"

### **Rerank前 (RRF分数)**
```
1. 玉清台 - RRF=0.0111
2. 女神 - RRF=0.0110
3. 石林 - RRF=0.0104
```

### **Rerank后 (CrossEncoder分数)**
```
1. 女神 - Rerank=0.9479 (94.8%) ← 语义最相关
2. 三清山 - Rerank=0.9227 (92.3%)
3. 玉清台 - Rerank=0.9212 (92.1%)
```

可以看到：
- Rerank完全改变了排序
- "女神"虽然RRF排名第2，但语义最相关，排到第1
- 分数都在0.9以上，说明都是高度相关的结果
