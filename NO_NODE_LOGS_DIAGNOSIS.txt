## 🎯 立即检查

### 检查1：是否有 [Background] 日志

**上传PDF后，在后端日志中搜索：** `[Background]`

#### 情况A：没有 [Background] 日志

**原因：** `asyncio.create_task` 未执行或立即失败

**可能性：**
1. 上传函数本身出错
2. `document_task_service.create_task()` 失败
3. Python异步任务未启动

**解决：**
在 `documents.py` 的第96行后添加日志：

```python
# 启动后台处理任务
logger.info(f"准备创建后台任务: {task_id}")
asyncio.create_task(
    process_document_background(task_id, file.filename, temp_file_path)
)
logger.info(f"后台任务已创建: {task_id}")
```

---

#### 情况B：有 [Background] Starting，但没有 [Workflow]

**原因：** `get_document_workflow()` 或 `workflow.process_document()` 失败

**可能性：**
1. workflow导入失败
2. process_document方法不存在
3. 执行出错但被try-catch捕获

**解决：**
检查是否有 `[Background] Document processing failed` 日志和错误堆栈

---

#### 情况C：有 [Workflow] Starting，但没有节点日志

**原因：** `astream()` 循环未执行或Graph编译失败

**可能性：**
1. Graph编译有问题
2. 状态初始化错误
3. astream循环立即退出

**解决：**
在 `workflow_service.py` 添加更多日志

---

## 🔧 快速修复方案

### 方案1：添加详细日志

**修改 `documents.py`：**

```python
# 在第96行前后添加
logger.info(f"═══ 准备启动后台任务 ═══")
logger.info(f"task_id: {task_id}")
logger.info(f"filename: {file.filename}")
logger.info(f"file_path: {temp_file_path}")

try:
    task = asyncio.create_task(
        process_document_background(task_id, file.filename, temp_file_path)
    )
    logger.info(f"后台任务创建成功，task: {task}")
except Exception as task_err:
    logger.error(f"创建后台任务失败: {task_err}")
    import traceback
    traceback.print_exc()
```

## 🎯 最可能的原因

根据经验，最可能的原因是：

### 1. asyncio.create_task未等待（70%可能）

Python的 `asyncio.create_task` 创建的任务是后台执行的，如果主进程退出或有未捕获的异常，任务会被丢弃。

### 2. workflow导入失败（20%可能）

`get_document_workflow()` 或 `workflow.process_document()` 导入失败，但错误被捕获。

### 3. 日志级别问题（10%可能）

logger配置的级别太高，INFO日志没有显示。
