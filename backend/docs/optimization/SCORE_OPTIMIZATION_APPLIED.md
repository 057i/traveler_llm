## 🔍 优化详情

### RRF (Reciprocal Rank Fusion) 工作原理

```python
# 不看分数，只看排名
dense_ranks: {doc1: 1, doc2: 2, doc3: 3}
sparse_ranks: {doc1: 5, doc2: 1, doc3: 10}

# RRF计算
score(doc) = 1/(k + rank_dense) + 1/(k + rank_sparse)

# k=60 时，文档1的分数
score(doc1) = 1/(60+1) + 1/(60+5) = 0.0164 + 0.0154 = 0.0318

# 然后归一化到 [0, 1]
```

**为什么RRF分数更高？**
- 不受原始分数尺度影响
- 基于排名融合，更稳定
- 归一化后分数分布更均匀
