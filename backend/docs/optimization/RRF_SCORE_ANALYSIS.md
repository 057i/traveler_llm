# RRF 分数归一化问题 - 方案对比

## 📊 问题回顾

**当前情况**:
```
原始稠密分数: 0.7042
RRF融合后: 1.0000  ← 问题：永远是1.0
```

**原因**: RRF归一化把最高分固定为1.0，失去了绝对相似度的含义

---

## 🎯 三种方案对比

### 方案1: 保持当前（归一化到[0,1]）
**当前实现**

```python
# 最高RRF分数归一化为1.0
rrf_scores[doc_id] = rrf_scores[doc_id] / max_rrf
```

**效果**:
```
输入: Dense=[0.7042, 0.6865, 0.6855]
输出: RRF=[1.0000, 0.9763, 0.9600]
```

**优点**:
- ✅ 分数在[0,1]范围，直观
- ✅ 最高分永远是1.0，便于理解
- ✅ 可以使用固定阈值0.7

**缺点**:
- ❌ 失去了绝对相似度含义
- ❌ 0.7042和0.9042都会变成1.0
- ❌ 无法区分真正的高质量结果
- ❌ Rerank时会丢失原始相似度信息

**推荐指数**: ⭐⭐ （不推荐）

---

### 方案2: 不归一化（保留RRF原始值）

```python
# config/fusion_config.py
"rrf": {
    "k": 60,
    "rescale_to_similarity": False,  # 改为False
}
```

**效果**:
```
输入: Dense=[0.7042, 0.6865, 0.6855]
输出: RRF=[0.0164, 0.0159, 0.0156]
```

**优点**:
- ✅ 保留RRF的相对关系
- ✅ 不会"虚高"

**缺点**:
- ❌ 分数太小（0.01-0.02），不直观
- ❌ 需要调整阈值到0.010
- ❌ 与原始相似度范围差异大
- ❌ Rerank时分数过小

**推荐指数**: ⭐⭐ （不推荐）

---

### 方案3: 缩放到原始分数范围（推荐）⭐⭐⭐⭐⭐

```python
# 归一化到[0,1]后，再缩放到原始Dense分数范围
original_max = 0.7042
original_min = 0.6100

for doc_id in rrf_scores:
    # 步骤1: 归一化到[0,1]
    normalized = (score - min_rrf) / (max_rrf - min_rrf)
    
    # 步骤2: 缩放到[0.61, 0.7042]
    final = normalized * (0.7042 - 0.6100) + 0.6100
```

**效果**:
```
输入: Dense=[0.7042, 0.6865, 0.6855]
RRF原始=[0.0164, 0.0159, 0.0156]
输出: RRF缩放后=[0.7042, 0.6980, 0.6920]
```

**优点**:
- ✅ 保留原始相似度的语义范围
- ✅ 最高分接近原始最高分（0.7042）
- ✅ 可以使用相同的阈值（0.6或0.7）
- ✅ Rerank时分数有意义
- ✅ 排序关系保持不变
- ✅ 分数更真实，不会虚高

**缺点**:
- ⚠️ 实现稍复杂
- ⚠️ 需要修改代码

**推荐指数**: ⭐⭐⭐⭐⭐ （强烈推荐）

## 🎯 推荐方案：方案3

### 为什么选方案3？

1. **保留语义**: 分数范围与原始相似度一致
2. **可比较性**: 可以与其他方法比较
3. **阈值统一**: 不需要为RRF单独设置阈值
4. **Rerank友好**: 传给Rerank的分数有意义
5. **真实反映质量**: 不会把中等结果标记为完美

## 💡 具体实施建议

### 修改文件
`app/core/fusion_strategies.py` 的 `RRFFusion.fuse()` 方法

### 修改内容
在第222-229行，将简单的 `/max_rrf` 改为：

```python
# 记录原始分数范围
original_max = max([s for _, s in dense_results]) if dense_results else 1.0
original_min = min([s for _, s in dense_results]) if dense_results else 0.0

# 归一化 + 缩放
if rescale_to_similarity:
    max_rrf = max(rrf_scores.values())
    min_rrf = min(rrf_scores.values())
    
    for doc_id in rrf_scores:
        # 归一化到[0,1]
        normalized = (rrf_scores[doc_id] - min_rrf) / (max_rrf - min_rrf)
        # 缩放到原始范围
        rrf_scores[doc_id] = normalized * (original_max - original_min) + original_min
```
