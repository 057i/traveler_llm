## 🎯 关键技术点

### 1. RAG搜索流程（8节点）

```
查询理解 → 并行检索(Milvus+Neo4j) → RRF融合 
→ 置信度检查 → Tavily补充 → RRF融合2 
→ Rerank精排 → LLM润色
```

### 2. 融合策略优化

**问题**: 加权平均稀释高分
```python
# 原始（有问题）
fused = 0.9 × 0.72 + 0.1 × 0.35 = 0.6505 ❌
```

**解决**: RRF + 分数缩放
```python
# RRF基于排名
rrf = 1/(60+rank_dense) + 1/(60+rank_sparse)

# 缩放到原始范围（关键！）
final = normalized × original_max = 0.72 ✅
```

### 3. Redis缓存架构

**数据结构设计**:
- List: 消息列表（有序）
- Hash: 消息详情、进度信息
- 自动过期: 7天（聊天）、1小时（进度）

**核心优势**:
- 内存级速度
- 自动过期清理
- 支持分布式
- 会话隔离
