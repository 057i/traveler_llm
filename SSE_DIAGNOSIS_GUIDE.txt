## 🔍 问题现象

1. 前端显示"无处理进度信息"
2. 后端没有打印节点日志
3. 前端没有调用SSE端点

## 📋 诊断步骤

### 步骤1: 检查上传响应

**操作:**
1. 打开浏览器开发者工具 (F12)
2. 切换到"网络"(Network)标签
3. 上传一个PDF文件
4. 找到 `/api/documents/upload` 请求
5. 查看"响应"(Response)标签

**期望看到:**
```json
{
  "task_id": "abc-123-def-456",
  "document_id": "abc-123-def-456",
  "filename": "xxx.pdf",
  "file_size": 12345,
  "status": "processing",
  "message": "文档上传成功，正在处理中"
}
```

**如果缺少 `document_id` 或 `task_id`:**
- 后端修改未生效，需要重启后端

---

### 步骤2: 检查前端taskId

**操作:**
1. 开发者工具切换到"控制台"(Console)
2. 上传PDF后，在控制台输入:
```javascript
console.log('documents:', documents.value)
```

**期望看到:**
```javascript
[
  {
    id: "abc-123",
    filename: "xxx.pdf",
    taskId: "abc-123-def-456",  // ← 应该有这个字段
    status: "processing"
  }
]
```

**如果 taskId 为空:**
- 前端关联逻辑有问题
- 检查 `handleUploadSuccess` 函数是否执行

---

### 步骤3: 检查SSE连接

**操作:**
1. 开发者工具-网络标签
2. 筛选器选择"EventStream"或"WS"
3. 查找 `/api/documents/progress/{task_id}` 请求

**期望看到:**
- 请求状态: 200 (持续连接)
- 类型: text/event-stream
- 响应: 持续接收数据

**如果没有这个请求:**
- taskId 为空
- WorkflowProgress组件未渲染

**如果请求404或500:**
- 后端路由问题
- task_id格式错误

---

### 步骤4: 检查后端日志

**操作:**
查看后端控制台输出

**期望看到的日志顺序:**
```
[INFO] 文件上传完成: xxx.pdf, 任务ID: abc-123
[INFO] [Background] Starting document processing for task: abc-123
[INFO] [Workflow] Starting: xxx.pdf
[INFO] [Workflow] Node executed: save_to_minio
[INFO] [Progress] minio: processing - 正在保存文件到存储... (15%)
[INFO] [Progress] minio: completed - 文件保存成功 (15%)
[INFO] [PDF] Starting PDF parse with MinerU
[MinerU] 开始解析文件: xxx.pdf
... (更多节点日志)
```

**如果只看到上传日志，没有 [Background] 或 [Workflow]:**
- 后台任务未启动
- asyncio.create_task 未执行
- process_document_background 函数有错误

**如果有 [Background] 但没有节点日志:**
- workflow.process_document() 未执行
- 或执行时出错但没有打印

---

### 步骤5: 检查Redis

**操作:**
```python
python -c "
from app.core.redis_client import get_redis_client

redis = get_redis_client()

# 查看所有document相关的键
keys = redis.keys('document:*')
for key in keys:
    print(f'{key}: {redis.get(key)}')
"
```

**期望看到:**
```
document:progress:abc-123: {"step": "minio", "status": "processing", ...}
document:event:abc-123: {"type": "start", ...}
```

**如果没有任何键:**
- workflow没有调用 _send_step_event
- Redis连接失败

## 🔧 常见问题和解决方案

### 问题1: taskId为空

**原因:** 后端响应缺少document_id

**解决:**
```bash
# 重启后端
cd backend
python main.py
```

---

### 问题2: 后台任务未启动

**原因:** asyncio.create_task 失败或被阻塞

**检查:**
```python
# 在 documents.py 的 upload_document 函数中添加日志
logger.info(f"创建后台任务: {task_id}")
asyncio.create_task(
    process_document_background(task_id, file.filename, temp_file_path)
)
logger.info(f"后台任务已创建")
```

---

### 问题3: workflow未执行

**原因:**
- workflow.process_document() 出错
- 或 astream() 未正确迭代

**检查:**
在 workflow_service.py 的 process_document 开头添加:
```python
logger.info(f"[Workflow] Starting: {filename}")
logger.info(f"[Workflow] Initial state: {initial_state}")
```

---

### 问题4: SSE连接失败

**原因:**
- CORS问题
- 前端代理配置

**解决:**
检查 backend/main.py 的 CORS配置:
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 🎯 快速诊断命令

### 检查后端路由
```bash
cd backend
python -c "
from app.api.documents import router
for r in router.routes:
    print(f'{list(r.methods)} {r.path}')
"
```

### 检查workflow配置
```bash
python -c "
from app.workflows.document_processing.workflow_service import get_document_workflow
w = get_document_workflow()
print('STEP_NAMES:', w.STEP_NAMES)
"
```

### 手动测试workflow
```bash
python -c "
import asyncio
from app.workflows.document_processing.workflow_service import get_document_workflow

async def test():
    workflow = get_document_workflow()
    result = await workflow.process_document(
        task_id='test-123',
        filename='test.pdf',
        file_path='./data/uploads/test.pdf'
    )
    print(result)

asyncio.run(test())
"
```
