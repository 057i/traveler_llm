## 问题确认

**现象：** 完全没有 [Upload] 日志输出

**原因：** 前端修改没有生效（热重载失败或浏览器缓存）

---

## ✅ 解决方案

### 方案1: 重启前端开发服务器（推荐）

```bash
# 1. 停止当前前端服务
按 Ctrl+C

# 2. 进入前端目录
cd frontend

# 3. 重新启动
npm run dev

# 4. 等待编译完成
看到 "ready in Xms" 提示

# 5. 强制刷新浏览器
按 Ctrl+Shift+R
```

---

### 方案2: 清除浏览器缓存

```
1. F12 打开开发者工具
2. 右键点击地址栏刷新按钮
3. 选择 "清空缓存并硬性重新加载"
```

---

### 方案3: 检查前端是否在运行

```bash
# 确认前端进程
npm run dev

# 应该看到类似输出：
  VITE v4.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

---

## 🧪 重启后测试步骤

### 1. 确认修改生效

**访问页面源代码：** 按 Ctrl+U

**搜索：** `console.log('[Upload] Response'`

**应该能找到：**
```javascript
console.log('[Upload] Response:', response)
```

如果找到了，说明代码已更新。

---

### 2. 上传PDF测试

1. 访问 http://localhost:5173/documents
2. F12 打开控制台
3. 上传PDF
4. 立即查看 Console

**应该看到：**
```javascript
[Upload] Response: {task_id: "...", document_id: "...", ...}
[Upload] Documents after fetch: [...]
```

---

### 3. 检查网络请求

**开发者工具 → Network 标签**

1. 上传PDF
2. 筛选 "documents"
3. 查看 `/api/documents/upload` 的响应

**响应应该包含：**
```json
{
  "task_id": "uuid-xxx",
  "document_id": "uuid-xxx",
  "filename": "xxx.pdf",
  "status": "processing"
}
```
