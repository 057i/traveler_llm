## 💡 日志增强模板

为其他节点添加日志时，可以参考以下模板：

```python
async def node_function(self, state: MultiAgentState) -> MultiAgentState:
    """节点功能"""
    logger.info("=" * 80)
    logger.info(f"[{self.role}] 🎯 开始执行任务")
    logger.info("=" * 80)
    
    # 打印输入
    user_query = state["user_query"]
    logger.info(f"[{self.role}] 📝 输入: {user_query}")
    
    # 执行任务
    logger.info(f"[{self.role}] 🔍 执行步骤1...")
    result = await self._do_something()
    logger.info(f"[{self.role}] ✅ 步骤1完成")
    
    # 打印结果
    logger.info(f"[{self.role}] 📋 结果: {result}")
    
    logger.info("=" * 80)
    logger.success(f"[{self.role}] ✅ 任务完成")
    logger.info("=" * 80)
    
    return state
```

## 🚀 使用建议

### 开发环境
- 设置日志级别为 `DEBUG` 查看所有日志
- 使用 `logger.debug()` 记录LLM原始响应（可能很长）

### 生产环境
- 设置日志级别为 `INFO`
- 关键步骤使用 `logger.info()`
- 成功使用 `logger.success()`
- 警告使用 `logger.warning()`
- 错误使用 `logger.error()`

---

## 📝 配置文件位置

日志配置在 `backend/main.py`:

```python
# 配置日志
logger.remove()
logger.add(sys.stdout, level=settings.LOG_LEVEL)
logger.add(
    settings.LOG_FILE,
    rotation="500 MB",
    retention="10 days",
    level=settings.LOG_LEVEL
)
```

修改 `config/settings.py` 中的 `LOG_LEVEL` 来调整日志级别：
- `DEBUG` - 显示所有日志
- `INFO` - 显示一般信息（推荐）
- `WARNING` - 只显示警告和错误
- `ERROR` - 只显示错误
