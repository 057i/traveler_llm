## 🎯 RRF使用场景

### 问题分析
- **问题**: RRF使用 `1/(k+rank)` 公式会压缩分数
- **现象**: 原始分数0.85 → RRF后0.029
- **原因**: 单一来源不需要融合，却强制做了RRF

### 解决方案
**只在需要融合多个来源时才使用RRF**

---

## 📊 RRF使用策略

### 场景1: 只有Milvus（不做RRF）✅
```python
输入:
  Milvus: 10条结果（分数0.85, 0.82, 0.79...）
  Tavily: 0条

策略:
  ⏩ 跳过RRF，直接保留Milvus原始分数

输出:
  分数: 0.85, 0.82, 0.79...（保持不变）
```

### 场景2: Milvus + Tavily（做RRF）✅
```python
输入:
  Milvus: 10条结果
  Tavily: 3条结果

策略:
  🔄 执行RRF融合两个来源

输出:
  分数: RRF融合后的分数（0.03左右）
  
注意: 虽然分数变小，但相对排序合理
```

---

## 🔍 RRF公式说明

```python
# RRF公式
RRF_score = Σ (1 / (k + rank))

# 示例
k = 60

来源1排名: 1, 2, 3
来源2排名: 5, 1, 2

景点A的RRF分数:
  来源1贡献: 1/(60+1) = 0.0164
  来源2贡献: 1/(60+5) = 0.0154
  总分: 0.0318
```

**特点**:
- ✅ 适合融合多个来源
- ❌ 会压缩分数（变成0.01-0.03范围）
- ⚠️ 单一来源不应使用

---

## 🎯 优化后的架构

```
Milvus检索 → 得到结果
    ↓
置信度检查: max_score < 0.5?
    ├─ 否 → 保留原始分数 ✅
    │        (不做RRF)
    │
    └─ 是 → Tavily补充
            ↓
            Milvus + Tavily
            ↓
            RRF融合 ✅
            (需要融合两个来源)
```
