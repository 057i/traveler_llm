## 🔍 问题现象

从网络请求截图看到：
- ✓ 有 documents.js（获取文档列表）
- ✓ 有 statistics（获取统计）
- ✗ 没有 progress 请求（SSE进度）

## 📋 原因分析

### WorkflowProgress组件没有渲染

**显示条件：**
```vue
<div v-if="row.taskId" class="progress-detail">
  <WorkflowProgress :task-id="row.taskId" ... />
</div>
```

**如果没有progress请求，说明：**
1. `row.taskId` 为空或undefined
2. 文档行没有展开
3. WorkflowProgress组件根本没有创建

---

## 🎯 立即检查步骤

### 步骤1：查看控制台日志

**在浏览器开发者工具 → 控制台标签**

搜索 `[Upload]` 关键字，应该看到：
```javascript
[Upload] Response: {task_id: "...", document_id: "...", ...}
[Upload] Documents after fetch: [...]
[Upload] Looking for document_id: ...
[Upload] Found doc: {...}
[Upload] Set taskId: ...
[Upload] Set status: processing
```

**如果没有这些日志：**
- handleUploadSuccess 没有执行
- 或者修改没有生效（需要刷新前端）

---

### 步骤2：手动查看documents数据

**在控制台执行：**
```javascript
// 查看所有文档
console.log('Documents:', documents.value)

// 查看taskId情况
console.log('TaskIds:', documents.value.map(d => ({
  id: d.id,
  filename: d.filename,
  taskId: d.taskId,
  status: d.status
})))
```

**期望看到：**
```javascript
[
  {
    id: "abc-123",
    filename: "xxx.pdf",
    taskId: "abc-123",  // ← 应该有值
    status: "processing" // ← 应该是processing
  }
]
```

**如果 taskId 为空：**
- response.document_id 和列表中的 id 不匹配
- find() 找不到文档
- doc.taskId 没有设置成功

---

### 步骤3：查看展开状态

**在控制台执行：**
```javascript

```

**期望看到：**
```javascript
["abc-123"]  // 新上传文档的ID
```

**如果是空数组：**
- 文档行没有自动展开
- 即使有taskId，也看不到进度组件

## 🔧 可能的问题和解决方案

### 问题1：前端修改没有生效

**原因：** 浏览器缓存了旧代码

**解决：**
```
Ctrl + Shift + R  (强制刷新)
或
Ctrl + F5
```

---

### 问题2：document_id 不匹配

**原因：** 后端返回的document_id和数据库ID不一致

**检查：**
```javascript
// 查看上传响应
// 应该在控制台已经打印，搜索 [Upload] Response

// 查看文档列表IDs
console.log('Document IDs:', documents.value.map(d => d.id))

// 对比是否匹配
```

**如果不匹配：**
后端需要确保 document_id 就是文档列表中的 id

---

### 问题3：状态立即变为completed

**原因：** 处理太快，刷新列表时已经完成

**解决：**
已在handleUploadSuccess中强制设置：
```javascript
doc.status = 'processing'
```

但需要确保这行代码执行了。

---

### 问题4：v-if条件还是太严格

**检查：** DocumentManager.vue 第101行

应该是：
```vue
<div v-if="row.taskId" class="progress-detail">
```

如果还是：
```vue
<div v-if="row.taskId && row.status === 'processing'" class="progress-detail">
```

说明修改没有保存或没有生效。

---

## ✅ 快速修复方案
