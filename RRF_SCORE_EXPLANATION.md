## 💡 RRF算法原理

### **RRF公式**
```
RRF_score = 1 / (k + rank)
```

其中：
- `k` = 60（常数，经验最佳值）
- `rank` = 排名（1, 2, 3, ...）

### **单路RRF分值范围**
```
Rank 1: 1 / (60 + 1) = 0.0164
Rank 2: 1 / (60 + 2) = 0.0161
Rank 3: 1 / (60 + 3) = 0.0159
Rank 10: 1 / (60 + 10) = 0.0143
Rank 60: 1 / (60 + 60) = 0.0083
```

**结论**：单路RRF分值天然就在 **0.008-0.016** 范围！

## 🎯 为什么RRF分值要这么小？

### **设计原因**

1. **基于排名，不基于绝对分值**
   - RRF只关心排名（1, 2, 3...），不关心原始分值（0.8 vs 0.3）
   - 这样可以公平地融合不同范围的分值

2. **避免某一路结果主导**
   - 如果用原始分值，Dense（0.7）会完全压倒Sparse（0.4）
   - RRF让每一路都有平等的发言权

3. **k=60是经验最佳值**
   - 太小（k=10）→ Top结果差距太大
   - 太大（k=100）→ Top结果差距太小
   - k=60在论文中被证明是最优值
