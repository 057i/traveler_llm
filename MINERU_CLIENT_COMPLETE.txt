```

    ↓
2. PUT file_url

    ↓
3. GET /api/v1/agent/parse/{task_id}
   → 轮询查询状态（pending/running/done/failed）
    ↓
4. GET markdown_url
   → 下载Markdown内容
    ↓

```

## 🔧 MinerU官方API说明

### API地址

**Base URL:** `https://mineru.net/api/v1/agent`

### 支持的参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| file_name | string | 必需 | 文件名 |
| language | string | "ch" | 语言（ch/en） |
| page_range | string | null | 页码范围，如"1-5" |
| enable_table | bool | true | 是否解析表格 |
| is_ocr | bool | false | 是否启用OCR |
| enable_formula | bool | true | 是否解析公式 |

### 任务状态

| 状态 | 说明 |
|------|------|
| waiting-file | 等待文件上传 |
| pending | 排队中 |
| running | 解析中 |
| done | 解析完成 |
| failed | 解析失败 |

## 📊 完整工作流

```
用户上传PDF
    ↓
1. MinIO存储节点 (15%)
   保存文件到本地
    ↓
2. PDF解析节点 (35%)
   调用 mineru_client.parse_by_file()

   [MinerU客户端执行:]
   a. 创建解析任务
      POST https://mineru.net/api/v1/agent/parse/file
      → 获取 task_id 和 file_url

   b. 上传文件到OSS
      PUT file_url
      → 上传PDF二进制数据

   c. 轮询等待结果（最多300秒）
      每3秒查询: GET .../parse/{task_id}
      → pending → running → done

   d. 下载Markdown
      GET markdown_url
      → 获取解析后的Markdown文本
    ↓
3. 文本分块节点 (50%)
    ↓
4. 实体提取节点 (65%)
    ↓
5. RAG向量化节点 (80%)
    ↓
6. GraphRAG节点 (95%)
    ↓
7. 完成节点 (100%)
```

## 🚀 使用步骤

### Step 1: 获取API密钥

1. 访问: https://mineru.net
2. 注册账号
3. 在控制台获取API密钥

### Step 2: 配置.env

```bash
cd backend
cp .env.example .env
# 编辑.env，添加:
MINERU_API_KEY=sk_your_actual_api_key_here
```

### Step 3: 测试客户端

```python
from app.core.mineru_client import get_mineru_client

client = get_mineru_client()
markdown = client.parse_by_file(
    file_path="test.pdf",
    language="ch",
    enable_table=True
)
print(markdown)
```

### Step 4: 启动后端

```bash
python main.py
```

### Step 5: 上传测试

1. 访问: http://localhost:5173/documents
2. 上传PDF文件
3. 观察实时进度和日志
