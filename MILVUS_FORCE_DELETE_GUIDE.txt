## 🚨 问题确认

**错误：** `expect 13 list, got 12`
**原因：** 旧collection还在，没有被删除或后端没重启

---

## ✅ 强制删除步骤

### 步骤1: 停止后端
```
在后端终端按 Ctrl+C
完全停止进程
```

### 步骤2: 执行删除命令
```bash
cd backend

python -c "from pymilvus import connections, utility; connections.connect('default', host='localhost', port=19530); print('Collections:', utility.list_collections()); utility.drop_collection('travel_destinations'); print('✓ 已删除'); print('剩余:', utility.list_collections())"
```

**预期输出：**
```
Collections: ['travel_destinations']
✓ 已删除
剩余: []
```

### 步骤3: 确认删除成功
```bash
python -c "from pymilvus import connections, utility; connections.connect('default', host='localhost', port=19530); print('现有collections:', utility.list_collections())"
```

**应该输出：**
```
现有collections: []
```

### 步骤4: 重启后端
```bash
python main.py
```

**观察日志，应该看到：**
```
[Milvus] Creating hybrid search collection with rich metadata: travel_destinations
[Milvus] Collection created with rich metadata support
[Milvus] Collection loaded: travel_destinations
```

### 步骤5: 重新上传PDF测试

## 🔍 如果还是失败

### 方案A: 改collection名称

**修改 `.env` 文件：**
```env
# 原来
MILVUS_COLLECTION_NAME=travel_destinations

# 改为
MILVUS_COLLECTION_NAME=travel_destinations_v2
```

然后重启后端，这样会创建新collection。

---

### 方案B: 重启Milvus服务

**如果使用Docker：**
```bash
docker restart milvus-standalone
```

**如果使用docker-compose：**
```bash
docker-compose restart
```

## ⚠️ 常见错误

### 错误1: 后端没有完全停止
**症状：** 删除后重启仍然报错
**解决：** 确保后端进程完全停止（检查任务管理器）

### 错误2: Milvus连接失败
**症状：** `Connection refused`
**解决：** 确保Milvus服务在运行
```bash
docker ps | grep milvus
```

### 错误3: 权限问题
**症状：** `Permission denied`
**解决：** 以管理员权限运行命令
