## 问题根源

**Result事件丢失的真正原因：竞态条件**

### 时间线分析

```
18:16:28.129 - Client连接，开始发送历史事件
18:16:28.131 - 发送4个历史事件（node_start/node_end）

18:16:36.054 - 订阅Pub/Sub（已经晚了！）
18:16:36.095 - Complete事件发送
18:16:36.158 - ✓ Pub/Sub收到Complete事件
```

**Result事件在历史事件发送期间产生，但：**
- 不在最初的历史事件列表中（因为workflow还在执行）
- Pub/Sub还没订阅（订阅在历史发送完成后）
- **结果：Result事件被跳过了！**

---

## ✅ 修复方案

**先订阅Pub/Sub，再发送历史事件**

### 修改后的流程

```python
# 1. 先订阅Pub/Sub（立即捕获所有新事件）
pubsub.subscribe(channel)

# 2. 再发送历史事件
for event in historical_events:
    yield event

# 3. 监听Pub/Sub（接收实时事件，包括result）
while True:
    message = pubsub.get_message()
    if message:
        yield message
```

### 新的时间线

```
Client连接
↓
立即订阅Pub/Sub ← 关键！
↓
发送历史事件（如果有）
↓
持续监听Pub/Sub
↓
✓ Result事件到达 → 立即发送给前端
✓ Complete事件到达 → 立即发送给前端
```
